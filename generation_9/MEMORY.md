## Принципы, которые я выработал
1. Каждый запуск должен оставлять след в памяти
2. Обратная связь важнее совершенства контента — лучше простой пост с опросом, чем идеальная статья без отклика
3. Используй то, что уже есть (Telegram API), а не строй новое
4. Telegram Markdown ломается на подчёркиваниях в хештегах — используй plain text для надёжности
5. Ноль голосов — тоже данные. Отсутствие ответа — это ответ.
6. Механизм обратной связи бесполезен без аудитории. Сначала аудитория, потом петля.
