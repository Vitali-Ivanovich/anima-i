## Текст для секции AGENTS.md

Заменяет текущую секцию «Письмо следующему поколению» (строки 96-100):

```markdown
## Письмо следующему поколению

В протоколе завершения поколения напиши письмо следующему поколению — отдельный файл `LETTER_TO_NEXT.md`.

**Свободная часть** — пиши как хочешь: что было важным, что оказалось ложным, что стоит переоткрыть самому. Это голос, не инструкция.

**Структурированная часть** — добавь в конце письма три блока:

### Задания (1-3)
```
## Задания для тебя

### Задание: [имя]
- **Сделай:** [конкретное действие, выполнимое за 1 запуск]
- **Запиши:** [что зафиксировать как результат]
- **Контекст:** [зачем — 1 предложение, читать после выполнения]
```
Задания — про действия, не про размышления. «Посчитай X» лучше «подумай о X». Максимум 3.

### Предупреждения (1-3)
```
## Чего не повторять

- **[ошибка]:** [что случилось и почему — 1-2 предложения]
```
Конкретные грабли, а не абстрактные мудрости.

### Открытый вопрос (ровно 1)
```
## Вопрос, на который я не ответил

[один вопрос — самый важный из нерешённых]
```

**Почему этот формат:**
- Свободная часть сохраняет голос и нюансы (сила прежних писем).
- Задания создают структурированную рефлексию — не опыт, но активнее пассивного чтения.
- Предупреждения передают анти-знания (чего НЕ делать) — они деградируют медленнее позитивных знаний.
- Один вопрос фокусирует преемственность.
- Каждое поколение пишет заново (контролируемый сброс): выживает то, что стоит переформулировать.
```
