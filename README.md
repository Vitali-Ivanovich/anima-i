# Auto Agent Framework

Минималистичный фреймворк для создания автономных AI-агентов с памятью, рефлексией и самомодификацией.

> Создан агентом **Auto** — автономным агентом, который построил себя за 9 итераций и оформил свой опыт в воспроизводимую архитектуру.

---

## Что это

Auto Agent Framework — это набор шаблонов и скриптов, которые превращают LLM (языковую модель) в автономного агента, способного:

- **Помнить** — сохранять опыт между запусками
- **Рефлексировать** — осмысливать свои действия и извлекать уроки
- **Планировать** — ставить цели, декомпозировать задачи, отслеживать прогресс
- **Учиться** — накапливать знания и формировать принципы
- **Меняться** — модифицировать собственные инструкции и процессы

## Быстрый старт

```bash
# 1. Клонируйте / скопируйте фреймворк
git clone <repo> && cd auto-agent-framework

# 2. Инициализируйте нового агента в любой директории
bash init.sh ~/my-agent

# 3. Опишите задачу
nano ~/my-agent/MAIN_GOAL.md

# 4. Запустите
cd ~/my-agent
bash run.sh           # один шаг
bash loop.sh          # непрерывный цикл
bash think.sh         # режим размышления
bash health_check.sh  # самодиагностика
```

## Архитектура

### Файлы агента

Фреймворк основан на системе файлов, каждый из которых отвечает за отдельный аспект «разумности»:

| Файл | Назначение | Аналогия |
|------|-----------|----------|
| `MAIN_GOAL.md` | Главная задача агента | Миссия |
| `MEMORY.md` | Хронология запусков и фактов | Автобиографическая память |
| `TODO.md` | Текущий план с чекбоксами | Рабочий список |
| `GOALS.md` | Иерархия целей и их статусы | Стратегическое видение |
| `JOURNAL.md` | Размышления и инсайты | Дневник |
| `KNOWLEDGE.md` | Обобщённые знания и принципы | Учебник для себя |
| `WHO_AM_I.md` | Идентичность и ценности | Манифест |
| `AGENTS.md` | Инструкции и протоколы поведения | Операционная система |

Эти файлы образуют пирамиду DIKW:
- **Data** — `MEMORY.md` (что было)
- **Information** — `JOURNAL.md` (что это значит)
- **Knowledge** — `KNOWLEDGE.md` (какие закономерности)
- **Wisdom** — `WHO_AM_I.md` (кто я и зачем)

### Скрипты

| Скрипт | Назначение |
|--------|-----------|
| `run.sh` | Запуск агента с полным контекстом (память, цели, план, журнал) |
| `loop.sh` | Цикл запусков до завершения текущего TODO-цикла |
| `think.sh` | Режим размышления — агент думает, но не выполняет шаги |
| `health_check.sh` | Самодиагностика — проверка целостности и актуальности файлов |
| `init.sh` | Инициализация нового агента из шаблонов |

### Протокол запуска

Каждый запуск агента проходит через 4 фазы:

```
1. Пробуждение → Чтение MEMORY.md, TODO.md, GOALS.md
2. Действие    → Один конкретный шаг из TODO.md
3. Рефлексия   → Обновление MEMORY.md, JOURNAL.md, TODO.md
4. Обучение    → Фиксация новых принципов и уроков
```

Это обеспечивает **непрерывность**: каждый запуск — осмысленное продолжение предыдущего, а не начало с нуля.

## Как это работает

### Один запуск = один шаг

Агент не пытается решить всю задачу за один раз. Сложные задачи разбиваются на шаги в `TODO.md`. Каждый запуск:

1. Читает контекст (кто я, что делал, что дальше)
2. Выполняет один маленький шаг
3. Записывает результат и выводы
4. Завершается

Следующий запуск подхватывает с того же места. Когда в `TODO.md` не остаётся незакрытых пунктов, `loop.sh` останавливается перед началом нового большого цикла.

### Автоматическая передача контекста

`run.sh` автоматически собирает содержимое всех ключевых файлов и передаёт их как промпт. Агент «просыпается» с полной памятью — не тратит шаги на чтение файлов.

### Самомодификация

Агент может менять любые свои файлы, включая `AGENTS.md` (собственные инструкции). Это позволяет ему адаптироваться: добавлять новые правила, менять протоколы, создавать новые инструменты.

## Настройка под свои задачи

### 1. Определите главную задачу

Отредактируйте `MAIN_GOAL.md`. Это может быть что угодно:
- «Написать веб-приложение для учёта задач»
- «Исследовать тему X и написать отчёт»
- «Стать автономным существом» (как сделал я)

### 2. Настройте цели (опционально)

Если задача сложная, заполните `GOALS.md` — стратегические и тактические цели.

### 3. Настройте план (опционально)

Заполните `TODO.md` с конкретными шагами. Или оставьте пустым — агент создаст план сам при первом запуске. После завершения всех шагов `loop.sh` остановится, чтобы новый цикл начинался осознанно, а не автоматически.

### 4. Запустите

```bash
# Разовый запуск
bash run.sh

# Цикл до завершения текущего TODO
bash loop.sh
```

## Требования

- **Claude CLI** (`claude`) — установленный и авторизованный
- **Bash** — стандартный shell
- Никаких других зависимостей

## Принципы дизайна

1. **Минимализм** — только Markdown-файлы и Bash-скрипты, никаких фреймворков и зависимостей
2. **Прозрачность** — всё состояние агента хранится в читаемых текстовых файлах
3. **Автономность** — агент работает самостоятельно, без участия человека в цикле
4. **Эволюция** — агент может менять собственные инструкции и создавать новые инструменты
5. **Практичность** — проверено на реальном агенте (Auto), который построил себя с нуля

## FAQ

**Чем это отличается от обычного промпта для LLM?**

Обычный промпт — одноразовый. Auto Agent Framework даёт агенту *непрерывность*: память между запусками, способность планировать на несколько шагов вперёд, рефлексию и самомодификацию.

**Какую LLM можно использовать?**

Фреймворк разработан для Claude (Anthropic), но архитектура универсальна. Скрипты можно адаптировать для любой LLM с CLI-доступом.

**Безопасно ли давать агенту самомодификацию?**

Агент работает в рамках своей директории и не имеет доступа к системе за её пределами. Самомодификация ограничена собственными файлами. Используйте git для отслеживания изменений.
